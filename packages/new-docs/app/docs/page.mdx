# Introduction

Composify is an open-source visual editor that works directly with your production components, designed to simplify Server-Driven UI development.

You register your existing React components, and Composify provides a drag-and-drop editor that anyone on your team—developers, marketers, designers—can use to compose pages visually. What you see in the editor is exactly what renders in production.

**Key characteristics:**

- **Drop-in integration**: No rewrites, no migration plans. You can start building with it today.
- **Zero lock-in**: Just a React library. You own your data and infrastructure.
- **No component drift**: Uses your actual production components, not a separate copy.
- **JSX-native**: Renders plain JSX strings, exactly like React. No custom DSL.

## Core Features

- **Component Registration**: Define what's editable by registering your React components with a simple catalog. Specify prop types, defaults, and validation—Composify handles the rest.
- **Direct JSX Rendering**: The renderer parses JSX strings and renders them using React's standard rendering. No abstraction layer, no proprietary format. Just JSX.
- **Visual Editor**: A drag-and-drop interface that works with your registered components. Non-developers can build pages without touching code, while developers maintain full control over the component catalog.

## How is Composify different?

There are other visual builders like [Plasmic](https://www.plasmic.app/), [Builder.io](https://www.builder.io/), and [Puck](https://puckeditor.com/). They're powerful tools, but come with trade-offs:

| Aspect | Traditional Builders | Composify |
|--------|---------------------|-----------|
| **Integration** | May require rewrites especially for child rendering | Works with existing components as-is |
| **Rendering** | Proprietary format or abstraction layer | Direct JSX rendering via React |
| **Infrastructure** | Often requires their hosting/backend | Bring your own backend, or use optional cloud |
| **Transparency** | Black-box rendering | Fully open-source, readable codebase |

Composify is built for teams that want visual editing capabilities without rebuilding their component library or coupling to a proprietary platform.

## When to use Composify

Composify works best when you:

- Already have a component library you want to reuse
- Want to enable non-developers to build pages without waiting on engineering
- Need server-driven UI capabilities for instant updates without redeployment
- Want to avoid vendor lock-in and maintain full control over your stack
- Need to test layouts or run A/B experiments with different page structures
